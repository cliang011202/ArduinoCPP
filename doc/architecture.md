# 系统架构设计说明

## 设计原则
- 高实时任务与低实时任务解耦
- 控制逻辑采用周期任务
- 控制参数通过消息机制传递
- 硬件资源由单一模块管理

## 任务划分
| 任务             | 优先级 | 职责         |
| ---------------- | ------ | ------------ |
| MotorControlTask | 高     | 周期控制电机 |
| CommandTask      | 中     | 串口命令解析 |
| LoggerTask       | 低     | 日志输出     |

## 通信机制
- Queue：传递控制命令
- Semaphore：保护串口资源
- EventGroup：系统状态同步

## 实时性分析
- 电机控制周期：10ms
- 控制任务不被串口和日志阻塞
- 日志系统可延迟执行
